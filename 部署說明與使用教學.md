## 部署說明與使用教學

> By Chris Chou

## 一、系統架構

| 項目 | 內容 |
|------|------|
| 平台 | Google Apps Script（Web App） |
| 資料庫 | Google Sheets |

## 二、部署前準備

### 2.1 建立 Google Sheets

1. 在 Google Drive 新建一個試算表
2. 記下網址列中的試算表 ID（`/d/` 和 `/edit` 之間的那段字串）
   - 例：`https://docs.google.com/spreadsheets/d/`**`1ABC...XYZ`**`/edit`

### 2.2 建立 Google Apps Script 專案

1. 開啟 [script.google.com](https://script.google.com)
2. 點「新增專案」
3. 修改專案名稱為「餐盒訂購系統」

---

## 三、部署步驟

### 3.1 貼上程式碼

在 Apps Script 編輯器建立以下 5 個檔案：

| 檔案名稱 | 類型 | 說明 |
|---------|------|------|
| `Code` | Script（.gs）| 後端邏輯 |
| `Admin` | HTML | 管理員介面 |
| `Print` | HTML | 列印訂購單頁面 |
| `View` | HTML | 唯讀查詢介面 |
| `appsscript` | JSON（Manifest）| OAuth 設定 |

> **注意：** `appsscript.json` 需先在「專案設定」勾選「顯示 appsscript.json 資訊清單檔案」才能看到

### 3.2 第一次授權

1. 函數選單選 **`setup`** → 點「執行」
2. 跳出授權視窗 → 點「審查權限」→ 選擇學校帳號 → 點「允許」
3. 執行完畢後查看執行記錄，確認全部顯示 ✅

### 3.3 連結 Google Sheets

1. 函數選單選 **`connectSheet`** → 點「執行」會失敗（需要 Sheet ID）
2. 改成部署後在系統設定介面連結（見 3.5）

### 3.4 部署 Web App

1. 右上角「部署」→「新增部署項目」
2. 選擇類型：**網路應用程式**
3. 設定：
   - 執行身分：**我（部署者）**
   - 具有存取權的使用者：**nhsh.tp.edu.tw 中的所有人**（學校網域）
4. 點「部署」→ 複製 Web App 網址

### 3.5 初始設定

1. 開啟 Web App 網址
2. 進入「⚙️ 系統設定」
3. 在「Sheet 連結」欄貼上 Google Sheets ID → 點「🔗 連結」
4. 連結成功後，系統自動建立所有分頁

### 3.6 修復分頁欄位（重要）

1. 進入「⚙️ 系統設定」
2. 點「🔧 修復設定分頁」
3. 或在 Apps Script 編輯器執行 **`fixSetup()`**

這個步驟會：
- 補齊 `vendors` 所有欄位（訂購方式、聯絡電話、網址、匯款帳號等）
- 補齊 `buyers` 分機欄
- 補齊 `config` 所有設定 key
- 補齊訂單分頁的請購欄位

### 3.7 填入基本設定

進入「⚙️ 系統設定」依序填入：

**設定分頁：**
- 廠商（名稱、單價、訂購方式、電話、匯款帳號）
- 送餐地點
- 送餐時間
- 用途
- 處室
- 訂購人（依處室，含分機）
- 管理員帳號（填入有管理權限的 email）

**通知設定：**
- 管理員收件信箱
- 出納收件信箱
- 採購人姓名（顯示在訂購單上）
- Google Drive 資料夾 ID（用於存放訂購單 HTML）

## 四、日常使用說明

### 4.1 四張首頁卡片

| 卡片 | 意思 | 點擊動作 |
|------|------|---------|
| 📦 今天送餐 | 今天有幾筆訂單 | 篩選今日訂單 |
| ⚠️ 三天內需訂購 | 3個工作天內要送餐但還沒訂購 | 篩選待確認訂單 |
| 💰 待核銷 | 已送餐但款項未處理完畢 | 篩選待核銷訂單 |
| 📅 未來待訂購 | 3天後以後的待確認訂單 | 篩選未來訂單 |

---

### 4.2 新增訂單

1. 點「➕ 新增訂單」
2. 填入**請購系統資訊**（請購單號、請購日期）
3. 選處室 → 選訂購人（分機自動帶入）
4. 選廠商
5. 填送餐日期、時間、地點
6. 填葷食、素食數量、單價
7. 選用途、付款方式
8. 填備註（選填）
9. 點「✓ 建立訂單」

---

### 4.3 訂購流程（紙本廠商）

1. 進入「📬 待訂購」→「📋 紙本訂購」
2. 勾選要訂購的訂單
3. 點「📋 訂購並列印」
4. 畫面底部出現藍色提示列 → 點「🖨️ 開啟列印」
5. 瀏覽器列印頁面自動開啟 → 列印紙本送去廠商

**存檔：** 訂購時系統自動將訂購單 HTML 存到 Google Drive（依送餐日期命名）

---

### 4.4 訂購流程（電話／網路廠商）

1. 進入「📬 待訂購」→「📞 電話／網路訂購」
2. 點廠商電話連結直接撥打，或點網址連結
3. 確認訂購後點「✓ 已訂購」
4. 訂單狀態變為「已訂購」

---

### 4.5 核銷流程（現金付款）

```
已訂購 → 送餐 → 付現金 → 拿發票 → 填入發票資訊 → 已核銷
```

1. 進入「💵 零用金清單」
2. **準備現金區塊**：送餐日前查看即將到來的現金訂單，提前準備零用金
3. **待核銷區塊**：送餐後付款取得發票，點「🔍 詳情」確認，點「填發票」輸入發票資訊
4. 現金訂單填完發票直接變「已核銷」

---

### 4.6 核銷流程（轉帳付款）

```
已訂購 → 送餐 → 拿發票 → 填發票 → 待通知出納 → 建立匯款草稿 → 已核銷
```

1. 進入「💳 匯款清單」
2. **待填發票區塊**：點「🔍 詳情」確認訂單，點「填發票」輸入發票資訊
3. 填完後狀態變「待通知出納」
4. **待通知出納區塊**：勾選要匯款的訂單 → 點「💳 建立匯款草稿」
5. 到 Gmail 草稿匣確認信件後手動寄出給出納
6. 系統自動將訂單標記為「已核銷」

---

### 4.7 狀態流程總覽

**現金：**
```
待確認 → 已訂購 → 已核銷
```

**轉帳：**
```
待確認 → 已訂購 → 待通知出納 → 已核銷
```

---

### 4.8 重新列印訂購單

- **單筆：** 在訂單列表對已訂購的訂單點「🖨️」按鈕
- **批次：** 勾選多筆訂單 → 批次操作列點「🖨️ 批次重新列印」

---

### 4.9 View 介面（唯讀）

非管理員帳號開啟系統會看到唯讀介面，可依處室查詢訂單狀態，無法新增或修改。

## 五、系統設定說明

### 5.1 一般設定資料

進入系統設定後，所有新增/刪除操作都是**暫存**，需要按「💾 儲存所有設定」才會寫入 Sheet。

有未儲存變更時，頁面頂部會出現黃色提醒列。

### 5.2 廠商設定

每個廠商可設定：
- **訂購方式**：紙本 / 電話 / 網路（影響「待訂購」頁籤的分類）
- **聯絡電話**：電話訂購時顯示可撥打連結
- **網址**：網路訂購時顯示可點擊連結
- **匯款帳號**：建立匯款草稿時自動帶入

### 5.3 通知設定

| 設定項目 | 說明 |
|---------|------|
| 管理員收件信箱 | 每日提醒信寄送對象 |
| 出納收件信箱 | 匯款草稿預設收件人 |
| 採購人 | 顯示在紙本訂購單上 |
| 每日發送時間 | 每日提醒的時間點（0-23 點）|
| Google Drive 資料夾 ID | 訂購單 HTML 存放位置 |

### 5.4 排程觸發器

設定好通知時間後，點「安裝排程觸發器」啟用自動通知功能：
- **每日提醒**：今日送餐清單 + 三天後未訂購提醒
- **月末提醒**：未通知出納的轉帳訂單清單

## 六、Apps Script 工具函數

| 函數 | 說明 | 使用時機 |
|------|------|---------|
| `setup()` | 授權所有服務 | 首次部署 |
| `fixSetup()` | 補齊所有分頁欄位 | 更新版本後 |
| `createTestData()` | 建立 7 筆測試資料 | 測試系統 |

| `migrateOldData()` | 舊欄位格式遷移 | 大版本升級 |
| `fixDateFormat()` | 修正日期時間格式 | 格式異常時 |
| `testGetOrders()` | 測試訂單資料讀取 | 除錯用 |
| `diagFull()` | 完整系統診斷 | 除錯用 |

## 八、常見問題

**Q: 開啟系統顯示「載入失敗」**
A: 進入 Apps Script 執行 `setup()` 重新授權，再重新部署

**Q: 列印視窗沒有跳出**
A: 訂購後畫面底部會出現藍色提示列，點「🖨️ 開啟列印」即可

**Q: Drive 存檔失敗**
A: 確認系統設定中已填入 Google Drive 資料夾 ID，且資料夾有分享給部署者帳號

**Q: 新欄位（分機、請購單號）沒有顯示**
A: 執行 `fixSetup()` 補齊 Sheet 欄位

**Q: 廠商匯款帳號沒有儲存**
A: 填完帳號後需點「💾 儲存帳號」暫存，再點頁面底部「💾 儲存所有設定」才會寫入 Sheet
